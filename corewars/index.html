Introduction to Core Wars

Core Wars is a programming game created by D.G. Jones and A.K. Dewdney in 1984, where programs (called "warriors") battle in the memory of a virtual computer called MARS (Memory Array Redcode Simulator). This implementation serves as a platform for studying emergent behaviors, genetic algorithms, and competitive coevolution in artificial intelligence.

Theoretical Foundation

The Core Wars environment provides a unique testbed for several areas of computer science research:

  • Genetic Algorithms: Warriors evolve through mutation and crossover operations
  • Game Theory: Strategies emerge from competitive dynamics
  • Self-Modifying Code: Programs can alter their own instructions during execution
  • Artificial Life: Programs exhibit life-like behaviors (reproduction, competition, adaptation)

The Redcode Language

; Example: The classic Imp warrior MOV 0, 1 ; Copy instruction at address 0 to address 1 ; Example: Dwarf bomber ADD #4, 3 ; Add 4 to the value at offset 3 MOV 2, @2 ; Move instruction at offset 2 to indirect address JMP -2 ; Jump back 2 instructions DAT 0, 0 ; Data (kills processes that execute it)

Genetic Algorithm Implementation

Our AI agents use genetic algorithms to evolve warriors over generations:

  1. Initialize population with random or template-based warriors
  2. Evaluate fitness through combat simulation
  3. Select parents using tournament selection
  4. Apply crossover and mutation operators
  5. Replace population with offspring
  6. Repeat until convergence or maximum generations
Research Applications: This system can be used to study coevolution, the evolution of complexity, arms races in artificial systems, and the emergence of cooperative or parasitic behaviors in competitive environments.

Emergent Strategies

Through evolutionary pressure, several canonical strategies have emerged:

Replicators (Paper)

Create multiple copies of themselves, overwhelming opponents through sheer numbers

Bombers (Stone)

Systematically write DAT instructions across memory to kill enemy processes

Scanners

Search for opponents before attacking, improving accuracy but sacrificing speed

‚ö° Tip #2: Speed vs. Effectiveness

Meta Analysis

The tournament reveals interesting dynamics in the Core Wars ecosystem:

  • Imp Dominance: Despite its simplicity, the Imp often performs surprisingly well
  • Bomber Reliability: Stone and Dwarf variants show consistent performance
  • Scanner Precision: Effective against predictable opponents but vulnerable to fast movers
  • Replicator Chaos: Paper strategies create unpredictable battle dynamics
  • Evolutionary Advantage: AI-evolved warriors often discover non-obvious strategies
